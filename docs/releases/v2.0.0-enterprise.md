# Recall Enterprise v2.0.0 Release Notes

Release date: 2026-03-13

## Highlights
- Edge reliability hardened for agent media handling with atomic `.part` downloads, checksum-first promotion, and cleanup on stream/network failures.
- Release governance strengthened with strict acceptance validation for tag/version consistency, ISO sign-off dates, and placeholder-identity rejection.
- Combined backend+agent coverage gate (`>=85%`) and expanded regression suites for worker/tracing/system and agent resilience paths.

## Operational impact
- Reduces risk of corrupted media cache during intermittent connectivity.
- Raises confidence in release readiness by enforcing stronger, machine-checked sign-off quality.
- Improves maintainability posture via broader automated regression and coverage enforcement.

## Backward compatibility
- No breaking API path change introduced in this release family (`/api/v1` and `/api/public/v1` remain stable).
- Existing device fleets remain compatible; agent behavior is hardened around download failures and offline fallback handling.
